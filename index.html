About
A little context.

I spent five years as a bank officer at Canara Bank in India — managing operations, leading a team, and learning what it means when systems have to work without excuses.

Then I moved to Boston and earned my M.S. in Telecommunication Networks from Northeastern (3.71 GPA). That's where I fell in love with network automation — writing Python against live routers, templating configs with Jinja2, and watching Ansible provision an entire branch office in under 60 seconds.

These days I spend my time building NetDevOps workflows: multi-vendor topologies on Containerlab, CI/CD pipelines with GitHub Actions, and validation suites with pytest and NAPALM. I think networks should deploy themselves and prove they work.

Netaform

Evolving
Network Automation, Endlessly Evolving

A phased network-as-code portfolio on Containerlab. Deploys an 8-node multi-vendor topology (Arista cEOS, Nokia SR Linux, FRRouting) with eBGP, OSPF, and VLANs. Provisioned by 5 Ansible role-based playbooks with Jinja2 templates — zero-touch in under 60 seconds. Includes 7 Python NETCONF scripts, Maarpu (a config drift detector), and a GitHub Actions CI/CD pipeline running 40 pytest tests via NAPALM that block merges on failure. Dual-layer secrets with Ansible Vault + GitHub Secrets, plus Gitleaks scanning.
